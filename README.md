# AI 对话应用 - 技术文档

## 1. 项目概述 (What)

### 1.1 项目目的
本项目是一个基于 Next.js 的 AI 对话 Web 应用，用于对接扣子编程（Coze）的 API 服务，提供流畅的对话体验。

### 1.2 核心功能
- **前端对话界面**：简洁的聊天界面，支持实时消息展示
- **SSE 流式响应**：支持 Server-Sent Events 实现流式输出
- **API 代理**：后端代理层隐藏真实 API 地址，保护后端服务

---

## 2. 技术架构 (Why)

### 2.1 技术栈
| 层级 | 技术 |
|------|------|
| 前端框架 | Next.js 16 (App Router) |
| UI 样式 | Tailwind CSS 4 |
| 语言 | TypeScript |
| API 代理 | Next.js Route Handlers |

### 2.2 架构设计

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   用户浏览器    │────▶│  Next.js 前端   │────▶│  API 代理层     │
│                 │     │  (端口 3000)   │     │  /api/chat      │
└─────────────────┘     └─────────────────┘     └────────┬────────┘
                                                         │
                                                         ▼
                                              ┌─────────────────┐
                                              │  扣子编程 API   │
                                              │  (无鉴权)       │
                                              └─────────────────┘
```

### 2.3 为什么这样做

1. **代理层必要性**
   - 扣子编程 API 无鉴权机制，任何人可直接访问
   - 代理层可隐藏真实 API 地址
   - 未来可在此层添加访问控制、限流等安全措施

2. **选择 Next.js**
   - 全栈能力，前端 + API 一体
   - 静态/服务端混合渲染
   - 生态成熟，部署方便

3. **Tailwind CSS**
   - 原子化 CSS，体积小
   - 开发效率高
   - 现代化 UI 构建

## 3. 实现细节 (How)

### 3.1 项目结构

```
.
├── .env.local              # 本地环境变量
├── app/
│   ├── api/
│   │   └── chat/
│   │       └── route.ts    # API 代理路由
│   ├── globals.css         # 全局样式
│   ├── layout.tsx          # 根布局
│   └── page.tsx           # 对话页面
├── .github/
│   └── workflows/
│       └── build.yml       # CI 构建流程
├── DEPLOYMENT.md          # 部署配置说明
└── package.json
```

### 3.2 环境变量

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `NEXT_PUBLIC_API_URL` | 是 | 扣子编程 API 地址 |
| `API_KEY` | 否 | API 密钥（如有） |

### 3.3 API 代理实现

代理路由位于 `app/api/chat/route.ts`，实现以下功能：
- 接收前端请求
- 添加必要的请求头（Content-Type、Authorization）
- 转发请求到扣子编程 API
- 流式返回响应（SSE）

### 3.4 前端对话实现

页面位于 `app/page.tsx`，主要功能：
- 消息列表展示
- 用户输入处理
- SSE 流式响应解析
- 加载状态显示
- 自动滚动到底部

## 4. 风险分析

### 4.1 安全风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| API 地址泄露 | 中 | 当前通过环境变量管理，生产环境可加访问限制 |
| 无访问控制 | 高 | 建议添加简单的访问控制或鉴权 |
| 请求频率无限制 | 中 | 建议添加限流中间件 |

### 4.2 部署风险

| 风险 | 说明 |
|------|------|
| 国内备案 | 阿里云/腾讯云国内节点需 ICP 备案 |
| 函数超时 | SSE 长连接可能导致函数超时，需配置足够超时时间 |
| 冷启动 | Serverless 首次调用可能有延迟 |

### 4.3 运维风险

- 扣子编程 API 不可用时服务中断
- 无监控告警，需自行配置

## 6. 快速开始

### 6.1 本地开发

```bash
# 安装依赖
npm install

# 配置环境变量
# 编辑 .env.local

# 启动开发服务器
npm run dev
```

访问 http://localhost:3000

### 6.2 构建生产版本

```bash
npm run build
npm run start
```

### 6.3 代码检查

```bash
npm run lint
```
